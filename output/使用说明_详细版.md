### 📋 Sheet 1：笔记列表 —— 搜到的所有笔记的基础信息

**👀 这张表是什么：**

每条搜索关键词搜出来的前 4 条笔记，共 47 条。这是最原始的数据底表，记录了每条笔记是谁发的、多少赞、什么时候发的。

**📊 每一列的含义：**

| 列名 | 类型 | 详细说明 | 举例 |
|---|---|---|---|
| 序号 | 数字 | 按组别排序后的编号，1~47 | `1` |
| 组别 | 文字 | 这条笔记属于哪个关键词组，共 4 组 | `🍹 口味上新` |
| 笔记ID | 字母+数字 | 小红书给每条笔记的唯一 ID，可以用来去重 | `6a4e1f01000...` |
| 标题 | 文字 | 笔记的完整标题，也是小红书 SEO 的核心字段 | `已严肃品鉴古茗新品。。。` |
| 作者 | 文字 | 发布这条笔记的用户昵称 | `一起烤地瓜🍠` |
| 点赞数 | 数字 | 这条笔记被多少人点赞了（代表热度） | `7886` |
| 发布日期 | 文字 | 笔记发布时间，部分笔记可能没有这个字段 | `2026-07-12` |
| 来源关键词 | 文字 | 我们搜的哪个关键词搜出了这条笔记 | `奶茶新品` |
| 链接 | URL | 点开可以直接跳转小红书原笔记 | `https://www.xiaohongshu.com/...` |

**🔧 怎么操作：**

1. **按组别筛选**：点「组别」列头上的下拉小三角 → 勾选你想看的组（比如只勾 ⚠️踩雷避坑）→ 点确定。这样只看某一个组的所有笔记。
2. **按热度排序**：点「点赞数」列头的下拉箭头 → 选择「降序」→ 笔记会从点赞最多的排到最少的，一眼看到最火的内容。
3. **去原帖看看**：直接点「链接」列的链接，浏览器会打开小红书原笔记页面（前提是你登录了小红书）。
4. **搜索特定品牌**：Ctrl+F → 输入「喜茶」「古茗」「霸王茶姬」等品牌名 → Excel 会在当前 Sheet 里搜索包含这些词的笔记。
5. **复制去重后的标题列表**：选中「标题」整列 → Ctrl+C → 粘贴到 Word 或记事本里，可以拿去做关键词词频分析。

**💡 小提示：**
- 「减脂期喝什么」只搜到 3 条而非 4 条，说明这个词下的内容去重后可用量偏少，后续做内容策划可以重点关注这个词。
- 「组别」列已经帮你按四大组排好序了，同一组内的笔记按点赞数降序排列。

### 💬 Sheet 2：评论明细 —— 420 条评论的完整记录

**👀 这张表是什么：**

对热度最高的 10 条笔记，每条抓了 30 条评论（含子回复）。最终去噪后得到 420 条有效评论。这是最核心的消费者洞察来源——**用户到底在说什么，全在这张表里**。

**📊 每一列的含义：**

| 列名 | 类型 | 详细说明 | 举例 |
|---|---|---|---|
| 组别 | 文字 | 该评论所属笔记的组别 | `⚠️ 踩雷避坑` |
| 楼层ID | 文字 | M1/M2/M3…表示主评论编号，M 后面的数字相同说明是同一条主评论下的讨论 | `M3` |
| 评论类型 | 二选一 | **主评论**（直接评论笔记，展开讨论的第一层）或 **子级回复**（回复别人的评论，也就是「楼中楼」） | `主评论` |
| 评论文本 | 文字 | 评论的实际内容，已过滤灌水噪音 | `减脂期喝这个真的不会胖吗求解答` |
| 点赞数 | 数字 | 这条评论有多少人点赞 | `23` |
| 发布时间 | 文字 | 评论发布的时间 | `2小时前` |
| 被回复人 | 文字 | 如果是子回复，这条是回复谁的 | `茶歇语` |
| 来源笔记 | 文字 | 这条评论出自哪条笔记（对应 Sheet 1 的标题） | `瘦了10斤！低卡奶茶合集` |
| 笔记链接 | URL | 点开直接跳转笔记详情页 | `https://www.xiaohongshu.com/...` |

**🎨 颜色标记：**

- **🟣 浅紫色底的行 = 主评论**（用户直接对笔记发表的意见）
- **⬜ 白色底的行 = 子级回复**（用户在楼中楼里互相回复）

这个颜色设计让你滚动表格时**一眼就能区分一级评论和楼中楼**，不用死盯着「评论类型」那列看。

**🔧 怎么操作：**

1. **只看吐槽**：筛选「组别」= ⚠️ 踩雷避坑 → 所有踩雷相关评论全展示出来，看看消费者讨厌什么产品/口味/品牌。
2. **只看主评论**：筛选「评论类型」= 主评论 → 过滤掉楼中楼，只剩 94 条精华观点。
3. **找热评**：「点赞数」列降序排列 → 找到最受认同的评论，这些是用户共识。
4. **品牌搜索**：Ctrl+F → 输入品牌名（比如「蜜雪冰城」「瑞幸」）→ 在全部评论中搜索该品牌被提及的情况。
5. **提取消费者原话**：选中「评论文本」整列 → Ctrl+C → 粘贴到 Word 或词云工具 → 做文本分析。
6. **按笔记看讨论**：筛选「来源笔记」→ 选一条感兴趣的笔记 → 看这条笔记下完整的讨论串。

**💡 小提示：**
- 子回复（326 条）远多于主评论（94 条），说明用户之间互动活跃，不是那种「发完就走」的僵尸评论区。
- 楼层 ID 相同的子回复属于同一条讨论链，可以还原对话上下文。

### 📊 Sheet 3：数据概览 —— 四个组别的汇总对比

**👀 这张表是什么：**

把四组数据的关键指标拉出来横向对比，一页看清全局。适合**写报告时引用数据**。

**📊 每一列的含义：**

| 列名 | 说明 | 怎么解读 |
|---|---|---|
| 组别 | 四大组名称 | 最后一行是合计 |
| 笔记数 | 该组收集了多少条笔记 | 理论值 12 条/组，实际可能有去重损失 |
| 评论总数 | 该组笔记下的所有评论（主+子） | 数字越大 = 该话题讨论越活跃 |
| 主评论 | 一级评论数量 | 代表有多少人主动发表观点 |
| 子回复 | 楼中楼评论数量 | 代表互动深度，数字高说明话题有争议性或共鸣强 |
| 平均点赞 | 该组笔记的平均点赞数 | 代表该类内容的平均热度水平 |
| 最高点赞 | 该组笔记中最火的一条有多少赞 | 看天花板在哪 |
| 最低点赞 | 该组笔记中最冷的一条有多少赞 | 看下限在哪 |

**🔍 怎么读出洞察：**

1. **对比「评论总数」**：哪组的数字最大 → 该话题最能引发讨论。通常 ⚠️ 踩雷避坑 的评论活跃度会偏高（大家吐槽欲旺盛）。
2. **对比「子回复 / 主评论」比值**：比值越高 → 该话题越容易引发争论或共鸣，用户不只发表观点，还会互相讨论。
3. **对比「平均点赞」**：哪组内容平均热度高 → 该方向的内容更容易获得流量。
4. **看「最高点赞」与「最低点赞」的差距**：差距大 → 该组内容质量参差不齐，好内容能爆但差内容没人看。

**📝 写报告时可以直接用：**

> "在四组关键词中，⚠️ 踩雷避坑组的评论互动最为活跃，子回复占比显著高于其他三组，说明负面话题在饮料品类中具有更强的社交传播力……"

### 🔀 Sheet 5：透视-组别×评论类型 —— 四大组横向对比矩阵

**👀 这张表是什么：**

将四大组的所有统计指标排成矩阵，一行一个组，一列一个指标。**这是最核心的对比分析表。**

**📊 列说明 & 分析思路：**

| 列名 | 怎么读 |
|---|---|
| 组别 | 行标签，浅橙/绿/蓝/紫底区分四组 |
| 笔记数 | 基础数据量，看是否有数据偏差 |
| 评论总计 | 互动总量指标 |
| 主评论 | 主动表达意愿指标 |
| 子回复 | 互动深度指标 |
| 平均点赞 | 内容平均质量指标 |
| 最高点赞 | 内容天花板指标 |
| 最低点赞 | 内容下限指标 |
| 平均每笔记评论 | **关键效率指标** —— 每条笔记平均能引发多少条评论 |

**🔍 核心分析框架：**

拿「平均每笔记评论」这个效率指标做核心对比：

- 如果 ⚠️ 踩雷避坑 的「平均每笔记评论」远高于 🍹 口味上新 → 说明**负面内容比新品推荐的互动效率更高**，消费者更愿意在吐槽内容下发表看法
- 如果 📸 视觉圈层 的「平均点赞」最高 → 说明**好看的探店图更容易获得点赞**（大家刷到图就会点，不需要细看）
- 如果 💚 健康诉求 的「子回复」比例最高 → 说明**健康话题容易引发讨论**（比如"这个真的低卡吗""我喝了反而胖了"之类的争论）

**📝 报告金句模板：**

> "从互动效率来看，负面/踩雷类内容的平均每笔记评论数（X 条）是正向推荐类内容（Y 条）的 Z 倍，表明饮料赛道的负面话题具有更强的用户参与驱动力。"

### 🔀 Sheet 6：透视-关键词明细 —— 12 个搜索词逐词拆解

**👀 这张表是什么：**

从组别粒度细化到关键词粒度，12 个搜索词各自的表现一目了然。**这是做内容选题策略的直接依据。**

**📊 列说明：**

| 列名 | 怎么读 |
|---|---|
| 组别 + 关键词 | 行标签，带组别前缀方便快速定位 |
| 组别 | 所属组别的底色 |
| 笔记数 | 该关键词实际收录的笔记数 |
| 评论总计 / 主评论 / 子回复 | 该关键词下的评论互动表现 |
| 平均点赞 | 该关键词搜出来的笔记平均有多少赞 |
| 最高点赞 | 该关键词下最火的一条笔记 |

**🔍 怎么发现机会：**

1. **找「笔记少但评论多」的关键词**：比如某个词只有 3 条笔记但评论却有 40+ 条 → 说明这个词的需求很精准但供给不足 → **内容蓝海机会**。
2. **找「平均点赞高」的关键词**：说明该方向的内容受欢迎程度高 → 做类似内容更容易获得流量。
3. **同组内 3 个关键词对比**：比如「低卡奶茶」「无糖饮料推荐」「减脂期喝什么」三个健康类关键词，哪个互动数据最好 → 优先用哪个词做内容标题。
4. **踩雷组关键词分析**：「奶茶避雷」「饮料踩雷」「最难喝饮料」三个词哪个最活跃 → 将来写避雷内容优先用那个词。

**📝 报告金句模板：**

> "在健康诉求组的三个关键词中，'减脂期喝什么'的互动数据表现最为突出，尽管收录量略低于其他两个关键词，但其笔记平均互动率更高，反映出精准减肥人群对饮品推荐内容的强烈需求。"

### 🔀 Sheet 7：透视-TOP笔记排行 —— 热度前 15 名

**👀 这张表是什么：**

点赞量最高的 15 条笔记的排行榜，含每条笔记的评论抓取数量。

**📊 列说明：**

| 列名 | 怎么读 |
|---|---|
| 排名 | 1~15，按点赞降序 |
| 标题 | 笔记标题，可以看出来「爆款标题」的套路 |
| 作者 | 博主名称 |
| 组别 | 属于哪一组，带底色 |
| 点赞数 | 这条笔记的热度 |
| 评论数 | 我们在这条笔记下实际抓到的评论数 |
| 来源关键词 | 搜什么词搜出来的 |

**🔍 怎么读出规律：**

1. **标题模式分析**：TOP 15 的标题有没有共性？—— 是否都带数字？都用了 emoji？都有「合集」「测评」「不重样」这类词？
2. **组别分布**：四个组都有上榜还是某一组霸榜？如果某组霸榜说明该类内容整体热度高。
3. **关键词 vs 热度**：哪些关键词更容易产出爆款笔记？
4. **评论数 vs 点赞数**：点赞多不代表评论多。有些笔记点赞很高但评论很少（大家看完就划走了），有些笔记点赞不多但评论爆炸（内容有争议性或实用性）。

**📝 报告金句模板：**

> "TOP 15 热门笔记中，自制教程类和低卡合集类内容占据了主要席位，反映出用户对'可操作性强'和'信息密度高'的内容有更强的互动意愿。"

## 🛠️ Excel 操作速成指南

### 1. 筛选（最常用的操作）

**干嘛的：** 只显示你想看的数据，隐藏不相关的行。

**怎么操作：**
1. 点表头单元格右边的 **▼ 下拉小三角**
2. 在弹出的菜单里**勾选**你想看的内容，**取消勾选**不想看的
3. 点「确定」
4. 取消筛选：点下拉 → 选「全部」或「清除筛选」

**实用场景：**
- 只看某一组的数据 → 在「组别」列筛选
- 只看主评论 → 在「评论类型」列筛选
- 只看点赞 > 100 的 → 在「点赞数」列选「数字筛选 → 大于」

### 2. 排序

**干嘛的：** 让数据从大到小或从小到大排列。

**怎么操作：**
1. 点表头下拉 → 选「降序」（从大到小）或「升序」（从小到大）
2. 也可以选中整列 → 点顶部菜单栏「数据 → 排序」

**实用场景：**
- 「点赞数」降序 → 最火的排最前面
- 「评论数」降序 → 讨论最多的排最前面

### 3. 冻结窗格（已帮你设好）

**干嘛的：** 往下滚的时候表头不会消失，始终能看到每一列叫什么。

所有透视 Sheet 都已经帮你冻结了首行，只管滚就行~

### 4. 搜索（Ctrl+F）

**干嘛的：** 在所有 Sheet 或当前 Sheet 里搜索关键词。

**怎么操作：**
1. Ctrl+F → 输入关键词（品牌名、产品名、任何你想找的词）
2. 点「选项」可以设置搜索范围（当前 Sheet 还是整个工作簿）
3. 点「查找全部」可以看到所有匹配结果

**实用场景：**
- 搜「喜茶」→ 看所有提到喜茶的笔记和评论
- 搜「好喝」→ 看所有正面评价
- 搜「太难喝」→ 看所有负面评价
- 搜「热量」「卡路里」→ 看健康相关的讨论

### 5. 复制数据到别处

**干嘛的：** 把 Excel 数据拿来放进 Word、PPT、或者在线词云工具。

**怎么操作：**
- 选中某一列 → Ctrl+C → 粘贴到别处
- 选中整个区块 → Ctrl+C → 粘贴到 PPT（保留格式）
- 选中某些行 → Ctrl+C → 粘贴到 Word（只保留文本：右键 → 粘贴选项 → 只保留文本）

### 6. 条件格式（给数字加色阶）

**干嘛的：** 让数字列自动显示颜色深浅，热数据一眼识别。

**怎么操作（以「点赞数」为例）：**
1. 选中「点赞数」整列
2. 顶部菜单「开始 → 条件格式 → 色阶」
3. 选一个颜色方案（推荐绿-白-红或蓝-白-橙）
4. 点赞越多颜色越深，热度分布一目了然

## 📋 常见分析场景 Recipe

### 🍽️ Recipe 1：品牌舆情分析

> 想知道某个品牌在小红书饮料讨论中的口碑？

1. 打开 Sheet 2「评论明细」
2. Ctrl+F → 输入品牌名（如「蜜雪冰城」）
3. 点「查找全部」→ 定位所有提到该品牌的评论
4. 逐条阅读，标记正面/负面/中性
5. 从 Sheet 1「笔记列表」看哪些笔记提到了该品牌，点赞量如何

### 🍽️ Recipe 2：内容选题决策

> 想做一期小红书饮料内容，不知道做什么方向？

1. 打开 Sheet 6「透视-关键词明细」
2. 找到「评论总计 / 笔记数」比值最高的关键词
3. 再对照 Sheet 7「TOP笔记排行」看该方向的热门标题长什么样
4. 从 Sheet 8「评论焦点」中该组的 TOP 评论里找用户最关心的问题
5. 选题 = 高互动关键词 + 热门标题模式 + 用户真实痛点

### 🍽️ Recipe 3：消费者需求洞察

> 饮料消费者最在乎什么？

1. 打开 Sheet 8「透视-评论焦点」
2. 读 TOP 20 热评，提取关键词和观点
3. 把热评按主题分类（比如：口感/价格/热量/包装/购买便利性/品牌偏好）
4. 统计每个主题出现的频次
5. 频次最高的主题 = 消费者最在乎的维度

### 🍽️ Recipe 4：负面舆情预警

> 有什么产品/品牌正在被大规模吐槽？

1. 打开 Sheet 2「评论明细」
2. 筛选「组别」= ⚠️ 踩雷避坑
3. 筛选「评论类型」= 主评论
4. 逐条阅读，提取被吐槽的具体产品和品牌
5. 统计出现频次，频次高 + 点赞多 = 高风险舆情

### 🍽️ Recipe 5：报告数据提取

> 要写一份饮料赛道分析报告，需要结构化数据：

1. 宏观数据从 Sheet 3 / Sheet 5 提取（组别对比数据）
2. 爆款案例从 Sheet 7 提取（TOP 笔记标题和关键词）
3. 消费者原声从 Sheet 8 提取（热评原文引用）
4. 品牌数据从 Sheet 2 Ctrl+F 搜索提取（品牌提及统计）

## 🧹 数据质量 & 局限性说明

### ✅ 已经帮你处理过的

- **灌水评论过滤**：以下类型的噪音已自动清除——
  - 「蹲」「码」「打卡」「1」「滴滴」「前排」等纯灌水
  - 纯标点、纯数字的评论
  - 「求链接」「在哪买」等纯求购信息（保留有实质内容的求购）
  - 过短的评论（少于 5 个字的）

### ⚠️ 需要知道的局限

| 局限 | 说明 | 影响 |
|---|---|---|
| 样本量有限 | 每关键词仅 4 条笔记，共 47 条 | 不能代表整个品类的全貌，适合定性分析不适合精确量化 |
| 时间截面 | 数据抓取于 2026 年 7 月 14 日单日 | 数据有时效性，一周后内容排名可能变化 |
| 搜索排序偏差 | 小红书搜索默认综合排序，不是纯热度排序 | 搜到的笔记有一定随机性 |
| 评论非全量 | 每笔记最多抓 30 条评论，非所有评论 | 可能遗漏一些长尾讨论 |
| 评论点赞量偏低 | 小红书评论点赞不像笔记点赞那么高 | 评论点赞的绝对值参考价值有限，主要看相对排名 |
| 部分笔记无评论 | 极小部分笔记可能因为风控原因评论抓取失败 | 已通过 fallback 机制降到最低 |
| 单账号视角 | 用同一个号搜索，看到的结果可能受账号画像影响 | 不同用户看到的热门内容可能不同 |

### 💡 基于这些局限的使用建议

- 这份数据的**最佳用途**：定性洞察（了解消费者在讨论什么）、内容方向参考、品牌舆情初筛
- **不太适合**：精确的市场份额计算、严格的统计学推断、跨时间趋势分析
- 如果要做更严谨的分析，建议：多次抓取取平均值、增加关键词覆盖量、组合多个账号的数据
